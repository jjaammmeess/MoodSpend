AfterBuy P5 — CloudKit 双机 / 离线 / 冲突 测试清单
================================================

前置条件
--------
- 两台 iOS 设备（或 1 真机 + 1 模拟器），登录同一 Apple ID，系统设置中 iCloud 已开启。
- Xcode 使用 Development 环境；CloudKit Dashboard 中容器 iCloud.JamesLiu.AfterBuy 可见。
- App 已从本分支安装；「我的 → 云同步」显示 CloudKit（非「未登录 iCloud」）。

A. 基础同步
-----------
[ ] A1 设备 A 新建一笔带 2 张照片的账单 → 设备 B 在 60s 内出现该账单与照片（首页 / 账单列表）。
[ ] A2 设备 B 修改该账单金额与备注 → 设备 A 刷新后显示新值（可切后台再回前台加速）。
[ ] A3 设备 A 删除该账单 → 设备 B 列表中该条消失（软删除，非重新出现）。

B. 自定义标签与设置
--------------------
[ ] B1 设备 A 新建自定义「品类」与「情绪」→ 设备 B 记一笔时可选择。
[ ] B2 设备 A 修改昵称/头像 → 设备 B 首页头像与昵称更新（回前台或等待导入完成）。
[ ] B3 设备 A 修改主题/语言/ pattern 规则 → 设备 B「我的」与界面语言一致。

C. 通知中心
-----------
[ ] C1 设备 A 产生情绪预警通知 → 设备 B 通知中心可见（允许数秒延迟）。
[ ] C1 设备 A 删除账单后，关联的「复盘」任务通知在 B 上被 prune（或标记只读残留可接受）。

D. 离线
-------
[ ] D1 设备 A 飞行模式记 3 笔 → 恢复网络后，设备 B 能收到全部 3 笔。
[ ] D2 设备 B 离线删除 1 笔 → 联网后设备 A 该笔消失。

E. 冲突（同一条同时编辑）
--------------------------
[ ] E1 两台同时打开同一笔账单，A 改金额为 100，B 改金额为 200，先后保存 → 最终仅保留一个金额，应用不崩溃、不丢其它字段（备注/类目等应尽量合并）。
[ ] E2 两台同时新建账单（不同 publicId）→ 两条均保留。

F. 首装 / 新设备
----------------
[ ] F1 全新安装、iCloud 有历史数据 → 首屏出现「正在恢复云端数据」，完成后账单齐全。
[ ] F2 全新安装、iCloud 无数据 → 60s 内进入空主页，不无限 loading。

G. 账号切换
-----------
[ ] G1 系统设置退出 iCloud 再登录 → 「我的」显示「未登录 iCloud」或恢复 CloudKit；数据不串号（旧账号数据不应与新账号混合，以 Apple 容器隔离为准）。

H. JSON 备份（与 iCloud 并存）
------------------------------
[ ] H1 合并恢复 v5 备份 → 不重复插入相同 publicId 的账单。
[ ] H2 合并恢复 v6 备份（含 customOption.publicId）→ 自定义标签不重复。

记录模板（失败时填写）
----------------------
用例 ID / 设备型号 / iOS 版本 / 期望 / 实际 / 截图或 Console 日志 / CloudKit Dashboard 是否有对应记录。
